1.1 Visi Permainan
Menciptakan pengalaman "zen" harian bagi pemain ekosistem Pi Network. Pemain dapat membuka game
selama 5-10 menit saat jeda istirahat, membuat beberapa keputusan strategis mengenai alokasi produksi,
lalu menutup game dengan perasaan puas bahwa "bengkel" mereka sedang bekerja untuk mereka.
1.2 Target Audiens
Target utama adalah pengguna Pi Browser yang terbiasa dengan rutinitas harian (menekan tombol petir
Pi). Game ini memanfaatkan kebiasaan tersebut dengan rutinitas paralel: masuk, panen, buat keputusan,
keluar. Game tidak menuntut pemain menatap layar berjam-jam.
1.3 Aturan Anti "Mindless Tapping"
Pi Forge secara sengaja mendesain bottleneck pada pembuatan keputusan (Crafting Slots) dan batas
tampung (Storage), sehingga bot tidak memiliki keunggulan kompetitif dibandingkan manusia yang
cerdas.

2. Art Direction & UI/UX Experience
2.1 Tema Visual: "Kawaii Cottagecore"
Game menggunakan palet warna pastel yang hangat. Tidak ada efek neon, tidak ada tema sci-fi, dan tidak
ada kesan gelap/berat. Visual diarahkan menyerupai buku cerita anak-anak, stiker jurnal, atau seni kertas
(papercraft).

4. Resource Management — Sistem Ore Tier
 ✦ SECTION UPDATED v2.0 — Ore Tier System Baru
Sistem ore diperbarui total dari 3 material menjadi 6 tier ore yang unlock secara progresif. Semakin lama
bermain dan semakin tinggi level bangunan, semakin langka dan berharga material yang bisa didapatkan.
Ini membuat resep makin bervariasi dan memperpanjang umur game secara signifikan.

item monetisasi "Ore Detector" (lihat Bab 8).
4.2 Offline Earnings Logic (Updated)
Formula kalkulasi offline:
Total Mined = Miner Level Rate * Detik Offline Actual Earned = min(Total Mined,
Space Left in Storage)
Jika Storage penuh, produksi otomatis berhenti. Storage base kapasitas dinaikkan dari 100 menjadi 150
untuk mengakomodasi variasi ore baru.

9. Sistem Anti-Curang
Sistem pengamanan diletakkan di komponen gameState.jsx (client-side validation):
clickTimestamps = clickTimestamps.filter(t => now - t < 1000); if
(clickTimestamps.length >= 8) return; // Block input
Array menyimpan histori klik dalam 1 detik. Jika 8+ klik terdeteksi dalam 1000ms, fungsi manualMine
melakukan early return. Ditambah Storage Cap dan Crafting Slot Cap memastikan klik masif tidak
menghasilkan koin tambahan.

15.8 Dampak ke Ekonomi Game
Monster Threat System dirancang sebagai coin sink yang organic:
 Pemain aktif: pilih tap (gratis, butuh waktu) → koin tidak terkuras
 Pemain sibuk: bayar Bodyguard (drain koin signifikan) → koin tidak menumpuk
 Pemain lalai: kehilangan ore (harus re-mine) → progres melambat alami
 Net effect: inflasi koin terkontrol, game bertahan lebih lama tanpa konten baru
